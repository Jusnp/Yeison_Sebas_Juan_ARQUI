# Guía de contribución

## Ramas
- `main`: producción
- `develop`: integración
- `feature/estudiante-funcionalidad`: trabajo individual

Ejemplos:
- `feature/juan-login`
- `feature/sebas-jwt`
- `feature/yeison-healthcheck`

## Commits semánticos
Formato:
`tipo(alcance): descripción`

Tipos:
- `feat`: nueva funcionalidad
- `fix`: corrección de errores
- `docs`: documentación
- `refactor`: refactorización

## Pull Requests
Todo cambio a `main` debe pasar por Pull Request aprobado.

## Reglas
- No trabajar directamente sobre `main`
- Documentar decisiones en `docs/adr/`
- Subir diagramas a `docs/c4/`
