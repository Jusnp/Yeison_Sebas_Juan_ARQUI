# Estrategia de Ramas

## Ramas principales
- `main`: código estable y listo para entrega
- `develop`: integración continua de avances

## Ramas de trabajo
Cada integrante debe trabajar en ramas derivadas de `develop`.

### Formato
`feature/nombre-funcionalidad`

### Ejemplos
- `feature/juan-drivers`
- `feature/yeison-c4`
- `feature/sebas-jwt`

## Flujo sugerido
1. Crear rama desde `develop`
2. Realizar cambios
3. Hacer commits semánticos
4. Abrir Pull Request hacia `develop`
5. Validar integración
6. Pasar a `main` cuando esté aprobado
