## Descripción del proyecto
Este proyecto propone el diseño de una solución web para la empresa **Green Mantenimientos y Servicios Ambientales**, con el fin de optimizar la gestión de solicitudes, programación, seguimiento y registro de servicios de mantenimiento, instalación y revisión.

## Objetivo general
Diseñar una arquitectura de software que permita soportar de forma eficiente, segura y mantenible la operación de una plataforma web para la gestión de servicios, historial de clientes, programación de técnicos, reportes y seguimiento de solicitudes.

## Organización del repositorio
La documentación del proyecto se encuentra organizada por semanas, de acuerdo con el cronograma definido en la asignatura:

- **Sem 1:** Kickoff y configuración de GitHub
- **Sem 2:** Drivers de calidad
- **Sem 3:** Priorización MoSCoW
- **Sem 4:** Patrón arquitectónico
- **Sem 5:** Modelo C4
- **Sem 6:** Inspección de riesgos
- **Sem 7:** SOLID y refactoring
- **Sem 8:** Seguridad JWT
- **Sem 9:** Docker y CI/CD
- **Sem 10:** Monitoreo y observabilidad
- **Sem 11:** Pruebas de estrés y caché Redis

## Flujo de trabajo
- `main`: rama estable
- `develop`: integración del proyecto
- `feature/nombre-funcionalidad`: ramas de trabajo individuales

## Convención de commits
Formato:

`tipo(alcance): descripción`

Ejemplos:
- `feat(auth): agregar login con JWT`
- `docs(sem-02): agregar drivers arquitectonicos`
- `fix(api): corregir validacion de usuarios`
- `refactor(service): separar responsabilidades de UserManager`
